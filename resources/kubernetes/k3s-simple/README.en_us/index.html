    K3s Simple Setup¶

    Due to the various versions of kubernetes references below will be focused on deploying to a k3s.io cluster. If you want information on how to spin up a cluster please check out the k3s.io link.

    Getting started¶

    Creating the project¶

    1. First we want to set up a project folder. Please create one and open it with your favorite text editor.
    2. Copy the contents of this directoy into your project directory.

    Spin-up¶

    This will create the resources!

    # We create the namespace first always before we spin up.
kubectl apply -f ./base/01-namespace.yml

# Now we can spin everything up.
kubectl apply -k .

    Spin-down¶

    This will remove all resources!

    kubectl delete -k .

    Breakdown¶

    Namespace¶

    This drives where the deployment will be filed under. Its best practices to segement each deployment in its own namespace. Namespace documentaion can be found here

    Kustomization¶

    The kustomization file is used to build a project and correlate data across multiple files. If you are looking for more details on documentation for kustomization click here.

    Persisted Volume Claim¶

    This is where your data is stored! It contains all data pertaining to your minecraft server and should be backed up! As a recommendation take a look at Velero. However, if you are looking for more information on persisted volume claims click here

    Deployment¶

    The deployment is what actually spins up your server and its related content. You can get more advanced with your deployment by adding additional environment variables, an admin panel, or other tools to help you administrait it. More information on deployments can be found here

    Join Service¶

    This provides the connection to your minecraft server! Its highly recommended to have a failover IP set as your external IP in a simple deployment. In a more advanced example you might want to have a proxy as your ingress if you are catering to multiple minecraft servers. More information on services can be found here
